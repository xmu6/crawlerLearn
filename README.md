# 🌱 网页爬虫

## 📖 学习日志

### 第一阶段：

**🎯 关键事件**：
- 第一个成功运行的爬虫，打印出了豆瓣电影Top10的标题
- 发现BeautifulSoup如此神奇，几行代码就能解析网页
- 成功把数据保存到CSV，感觉自己像个“数据矿工”

**🛠️ 技术焦点**：
- 基础的requests + BeautifulSoup / XPath组合
- 简单的正则表达式提取
- 文件读写操作

🤔 ：
> “爬虫原来这么简单！”



### 第二阶段：
**🎯 关键事件**：
- 第一次遇到403错误，网站竟然把我屏蔽了
- 发现有些页面需要登录，有些数据是JS动态加载的
- 遇到了第一个加密，第一次进行js逆向，第一次在f12中找源码，打断点

**🛠️ 技术焦点**：
- 学习添加headers和cookies，refer防盗链
- f12断点，不同断点的简单应用（某音）
- 摘要算法，base64编码，URL编码，非对称加密

🤔 ：
> “原来每个网站都像一座城堡，有的门好开，有的门上挂着‘非请勿入’。”



### 第三阶段：

**🛠️ 技术焦点**：

🤔 ：

## 📮 我想说

### 💡当前状态

这个仓库会保持现在的样子——不完美，但真实。暂停的核心原因并非兴趣丧失或畏惧难度，而是基于三点理性判断：

- 学习边际效应：在有限的时间内，进一步深入所需的精力投入（如深入研究分布式调度、高性能异步处理、复杂验证码破解等）与我的整体技术成长路径的优先级暂时不匹配。
- 知识应用的滞后性：没有迫切的、真实的业务场景驱动，高阶技能的习得效率较低，且难以巩固。
- 一半场景自动化已经够用，协议逆向又难又折磨。

> 由于工作和时间的原因，还有app逆向的部分没学，越学发现要学的越多，单也还没有接。
> 
>令人感慨。
